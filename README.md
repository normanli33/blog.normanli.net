# normanli.dev

A Hugo site — Essays, Lab, Digital Garden, Stream, About, Contact — with a warm, restrained
editorial design (light/dark, ⌘K search, sticky TOC, KaTeX + Mermaid, RSS, sitemap).

## Local development

Requires [Hugo extended](https://gohugo.io/installation/) 0.145+.

```bash
hugo server -D
```

Visit `http://localhost:1313`.

## Structure

```
content/        Markdown content — essays, lab, garden, stream, about, contact
layouts/        Templates: baseof + per-section list/single, partials, taxonomy
static/         CSS, JS, favicon, _headers, resume PDF
data/timeline.yaml   Career timeline shown on Home and About
archetypes/     `hugo new essays/my-post.md` etc. scaffold front matter per section
hugo.toml       Site config, menu, taxonomies, markdown/highlight settings
```

## Before you deploy

- Replace `static/files/norman-li-resume.pdf` with your real résumé.
- Update `baseURL`, `params.email`, `params.github`, `params.linkedin` in `hugo.toml`.
- Swap the sample essays/lab/garden/stream entries in `content/` for your own writing.

## Deploying to Cloudflare Pages

**Option A — Git integration (recommended)**

1. Push this project to a GitHub/GitLab repo.
2. In the Cloudflare dashboard: **Workers & Pages → Create → Pages → Connect to Git**, pick the repo.
3. Build settings:
   - Framework preset: `Hugo`
   - Build command: `hugo --gc --minify`
   - Build output directory: `public`
   - Environment variable: `HUGO_VERSION` = `0.164.0`
4. Save and deploy — every push to the main branch redeploys automatically.

**Option B — Direct upload with Wrangler**

```bash
npm install -g wrangler
hugo --gc --minify
wrangler pages deploy public --project-name=normanli-dev
```

## Notes on features

- **Search (⌘K)**: `static/js/search.js` fetches `/index.json` (generated by `layouts/index.json.json`
  from every Essay/Lab/Garden/Stream page) and filters client-side — no external service required.
- **Digital Garden cross-links**: related topics and "referenced in" backlinks on each Garden page
  are computed automatically from shared `tags` front matter — add a tag to a new essay and it
  shows up on the matching Garden note without any manual linking.
- **Math & diagrams**: set `math: true` or `mermaid: true` in a page's front matter to load
  KaTeX or Mermaid only where used, keeping other pages light.
- **Dark mode**: follows system preference by default; the header toggle stores an explicit
  choice in `localStorage` and sets `data-theme` on `<html>`.
